# Trac Cleanup Ledger

This ledger records Trac wiki pages that were inspected during the developer
documentation migration and should not become maintained repository
documentation. It is intended as a cleanup checklist for the repository import:
each row says whether the page should stay as historical Trac context or be
removed from the repository import. Removing an imported page does not delete
the original Trac record; that record remains subject to the archive policy
below. Any narrower follow-up is recorded in the row's reason without changing
that repository disposition.

If Trac itself is retired later, do not turn historical discussion pages into
current developer instructions. Preserve the original record in a static Trac
or website archive with its original page name and links where practical, move
durable decisions into maintained repository or website docs, and move still
open work into Trac tickets, replacement issues, or
`doc/development/todo.md` before retiring the source page.

Historical meeting, sprint, and event pages follow that archive rule. Keep the
original notes as dated project history, extract unresolved technical ideas into
current planning surfaces, and keep recurring workflow instructions in the
maintained developer documentation rather than maintaining meeting minutes as
living docs.
